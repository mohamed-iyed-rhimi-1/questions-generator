# .env File Note

The `.env` file is gitignored and should be created by copying `.env.example`:

```bash
cp .env.example .env
```

## Docker-Specific Configuration

When using Docker, the following environment variables are automatically configured:

### Database Connection
- **For Docker**: `DATABASE_URL` is overridden in `docker-compose.yml` to use `postgres` as the hostname (Docker service name)
- **For local development**: Use `localhost` as the hostname

### Ollama Connection
- **For Docker**: The backend uses `host.docker.internal:11434` to reach Ollama running on your host machine
- **For local development**: Use `localhost:11434`

### Important Notes
1. Ollama should be running on your host machine, not inside Docker
2. Run `ollama pull iKhalid/ALLaM:7b` (or your preferred model) before starting the services
3. The `.env` file contains sensitive configuration - never commit it to version control

## Default .env Content

Copy the content from `.env.example` and customize as needed. The default values work for most Docker setups without modification.
